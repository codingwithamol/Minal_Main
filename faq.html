Frequently Asked Questions ?

1. What is the difference between synchronous and asynchronous programming?
Synchronous code is executed line by line, while asynchronous code allows tasks to run in the background and doesn't block the main execution thread.
2. What are closures in JavaScript?
A closure is a function that retains access to its lexical scope even when executed outside of that scope.
3. What is the difference between == and === in JavaScript?
`==` compares values with type coercion, while `===` checks for strict equality (no type conversion).
4. What are promises and how do they work?
Promises are objects representing the eventual completion or failure of an asynchronous operation. They have `.then()` and `.catch()` methods.
5. What is the event loop in JavaScript?
The event loop handles asynchronous callbacks by managing the call stack and task queue to execute code non-blockingly.
6. How does memory management work in programming languages like C and Python?
C requires manual memory management using `malloc` and `free`, while Python uses automatic garbage collection.
7. Explain the concept of multithreading and concurrency.
Multithreading is running multiple threads simultaneously within a process, while concurrency is managing multiple tasks over the same period.
8. What is a deadlock and how can it be prevented?
A deadlock occurs when two or more processes wait indefinitely for resources held by each other. Prevention involves resource hierarchy, timeouts, or avoiding circular waits.
9. What are design patterns? Give examples.
Design patterns are reusable solutions to common problems. Examples include Singleton, Factory, Observer, and Strategy.
10. What is the difference between REST and GraphQL?
REST uses predefined endpoints and returns fixed data structures, while GraphQL lets clients specify exactly what data they need from a single endpoint.
